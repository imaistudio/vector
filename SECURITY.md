# Security Policy

## Supported Versions

Security fixes are handled on a best-effort basis for the current default branch.

## Reporting a Vulnerability

Please do not open a public issue for security vulnerabilities.

Instead:

1. Contact the repository owner privately through GitHub: https://github.com/xrehpicx
2. Include a clear description, impact, reproduction steps, and any suggested mitigation.
3. Share enough detail for the issue to be reproduced safely.

You can expect an acknowledgment after the report is reviewed. Response times are best-effort and may vary.

## Disclosure

Please give maintainers reasonable time to investigate and ship a fix before public disclosure.
